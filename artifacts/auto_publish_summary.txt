# Auto-Publish Implementation Summary

## Overview

This implementation enables AIConnect to automatically publish AI-generated media to user feeds with comprehensive safety controls, realtime event broadcasting, and moderation oversight.

## Architecture

### 1. Media Generation Pipeline
- **User Action**: Submit generation request via `/api/generate`
- **Backend**: Queue job in Redis; return jobId to client
- **Worker**: Process job (GPU model) → upload to MinIO S3
- **Callback**: Worker calls `/internal/posts/create` with generated media URLs

### 2. Auto-Publish Decision Logic
Located in `backend/src/routes/internal.js`:

```
Check AUTO_PUBLISH_MODE env var
├─ 'disabled' (default): All posts held for review
├─ 'trusted': Auto-publish only for trusted users (>30 days old, no flags)
└─ 'all': All posts auto-publish (high-risk)

Override if safety violation detected:
├─ Detection score < 0.8: Flag for review
├─ Provenance.flagged=true: Flag for review
└─ Policy keyword match: Auto-reject
```

### 3. Realtime Event Broadcasting
- **Publisher** (`backend/src/events/publisher.js`): In-memory event bus
- **WebSocket Server** (`backend/src/ws/server.js`): Socket.io with JWT auth
- **Frontend Client** (`frontend/src/services/realtime.js`): Subscribes to `post.created` events

Flow:
```
Worker → /internal/posts/create
→ postService.createPost()
→ publisher.publish('post.created', eventData)
→ WebSocket server emits to all connected clients
→ Frontend receives, prepends to feed UI
```

### 4. Safety & Moderation
See [docs/safety.md](../docs/safety.md) for detailed policy.

Key endpoints:
- `GET /api/moderation/queue` - List flagged posts
- `POST /api/moderation/:postId/action` - Approve/reject/escalate
- `POST /api/reports/create` - User reports
- `GET /api/posts/:postId/provenance` - Authenticity check

## Security

- **Bearer Token Auth**: All `/api/*` routes require JWT in `Authorization: Bearer` header
- **Internal Token Auth**: `/internal/*` routes require exact `INTERNAL_TOKEN` match
- **Password Hashing**: bcrypt with 10 rounds (see authController.js)
- **No Secrets in Code**: All sensitive values via environment variables
- **CORS**: Frontend origin validation in WebSocket server
- **Rate Limiting**: Placeholder middleware (configure in production)
